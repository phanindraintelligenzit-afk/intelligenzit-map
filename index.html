Guided tour: how a Digital Worker is produced

Walk the factory floor. Six stages from client need to scored, deployed FTE.

01

Shared runtime

The factory floor. LangGraph orchestration runs the agent loop. The tool framework registers every capability the worker can use. Memory persists across sessions. Observation captures every decision for audit.

LangGraphPostgresboto3MCP
02

Domain pack

The vertical specialisation layer. For cloud engineering: 5 KRAs (cost, security, compliance, performance, reliability), AWS tool set, policy guardrails. A new vertical reuses everything inside. Only this layer is new.

KRA definitionsAWS toolspolicy engine
03

Instantiate

From the client template: config, MCP connectors to client systems, data bindings. Each engagement is a new instance. Regeneron is live. smart Europe is scoped.

YAML configMCP connectorsdata isolation
04

Worker runs

Chandra executes. Claude Sonnet on Bedrock reasons across all 5 KRAs. Findings stream to the Streamlit dashboard. Terraform plans wait for human approval. PoC demoed live in Bangalore.

Claude SonnetBedrockStreamlitTerraform
05

Performance Index IN BUILD

The scoring layer. DPI reads 0-100 across 7 dimensions: Productivity, Quality, Execution, Governance, Risk, Validation, Cost. Formula: PI = (P x Q x 1.5E) + (G x 1.5R) + (C x V). Must read off real agent signals, not synthetic data. This is the frontier.

7 dimensionsreal signalscustomer-configurable
06

Deployed FTE + scorecard

The output. A productised, scored, sellable Digital FTE. Three commercial models: implement (build it), consult (advise), rent (ongoing worker-as-a-service). Closes the loop from client need to measurable outcome.

implementconsultrent
